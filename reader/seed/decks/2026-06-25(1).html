Long read · LAB deep dive · Grader · Harness · RL 06 / 08

How the win was won: a cheap-but-trusted judge, then RL

Applied Compute × Harvey·grader analysis · harness · full-parameter RL harvey.ai/blog ↗
  • The judge — why a small model? Reward is LLM-judged, so it must be trusted. Frontier consensus = GPT-5.5 xhigh + Opus 4.8 Max agreeing >95% on a 50-task / 2,500-criteria subset.
  • GPT-5 Mini held >97% alignment to that consensus (Nano ~95%, Haiku worse); top graders 98.9% self-consistent.
  • “Only for cost?” — yes, and cost is the game: RL grades thousands of rollouts every step, so grading dominates the bill. Mini + batching criteria = 40× cheaper at 16/call, 100× when all grouped — what makes high-compute RL affordable.
  • The harness: just read + grep over a sandboxed file set, hardened from live failures — cap tokens per call, repair malformed calls, better tool docs, plus compaction (90th-pct task ≈ 100k doc tokens; largest > 200k).
  • “The harness made better strategies available; RL taught the model to execute them.” The best result came from post-training inside the new harness — not the harness alone.

“Critique of the Agent Model” — where agency actually begins

Eric Xing et al·via @omarsar0 arxiv.org/pdf/2606.23991 ↗
  • A primary paper giving vocabulary to the field's most overloaded word, across five axes: goal · identity · decision-making · self-regulation · learning.
  • Its one sharp edge — agentic (behavior bolted on by an external harness) vs agentive (capability that lives inside the model itself).

“The Boundary Between Agentic and Agentive Systems”, never thought this paper would start from english lessons… the topic is actually quite good and i would love if someone can clarify the concepts like 'agent', or 'memory', or 'evolution'. but the specific content still hasn't exceed beyond the most common definitions about what an agent system has. maybe i read too quickly and omitted something.

Claude's read

The agentic/agentive split is the real contribution: agentic = behavior your harness bolts on; agentive = capability that lives in the model. By that line most of today's “agents” — tool-loops, memory-as-filesystem — are agentic. Yibo's right the rest restates common definitions, but the value isn't a taxonomy, it's a criterion: delete the harness — if the behavior vanishes, it was never agentive.

vLLM PR #46329 opens native NVFP4 KV cache on SM120/121 via FlashInfer — and keeps the flag off: “don't tell users it's supported yet; build the branches, serve --kv-cache-dtype nvfp4.”
Claude's read

In plain terms: the KV cache is the model's running memory of everything you've said this conversation. NVFP4 stores those entries in a 4-bit format native to Blackwell instead of 16-bit — same context, a quarter of the memory, so one card holds ~3.5× more tokens (here 5.1M → 18.1M). That's longer context and more concurrent users, near-free, if accuracy holds. The honesty is in the PR refusing to flip the “supported” flag before the kernels are proven — maturity you can't fake in a launch thread.
